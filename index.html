Binary Arithmetic

This section covers binary addition, subtraction, and two's complement arithmetic.

Binary Addition: To add two binary numbers, align them by their least significant bit (rightmost digit) and add each pair of bits, carrying over when the sum is 2 (10 in binary) or more. For example, 1 + 1 = 10, so you write 0 and carry 1 to the next column. Continue this process for all columns.

Binary Subtraction: Subtracting binary numbers is similar to decimal subtraction. If the top bit is smaller than the bottom bit, borrow 1 from the next column to the left. For example, 0 - 1 requires borrowing, making it 10 - 1 = 1, and the next column is reduced by 1.

Two's Complement: Two's complement is a method for representing negative numbers in binary. To find the two's complement of a binary number, invert all the bits (change 0 to 1 and 1 to 0) and add 1 to the result. This allows you to perform subtraction by adding the two's complement of a number.

Quiz Tips: For arithmetic questions, pay attention to whether the numbers are signed (using two's complement) or unsigned. Always check the number of bits (usually 8) and remember to handle carries and borrows correctly.

ASCII

This section explains the ASCII character encoding and how characters are represented as numbers.

What is ASCII? ASCII (American Standard Code for Information Interchange) is a character encoding standard that assigns a unique number to each character (letters, digits, symbols, and control codes). For example, the letter 'A' is 65, 'a' is 97, and '0' is 48.

Character to ASCII: To find the ASCII value of a character, look it up in an ASCII table or use programming functions (like ord('A') in Python). For quizzes, you may be asked to enter the decimal value for a given character.

ASCII to Character: To convert an ASCII value to a character, use an ASCII table or a function (like chr(65) in Python). For quizzes, you may be given a number and asked to write the corresponding character.

Quiz Tips: Remember that ASCII values for uppercase and lowercase letters are different. Control characters (values 0-31) are usually not included in quizzes.
You do not need to memorise ASCII codes for exams—an ASCII table will be provided if needed. The quiz is just for fun and practice!

Base Conversion

This section covers converting numbers between different bases, such as binary, decimal, and hexadecimal.

Decimal to Binary: Divide the decimal number by 2, recording the remainder. Continue dividing the quotient by 2 until you reach 0. The binary number is the remainders read from last to first.

Binary to Decimal: Multiply each bit by 2 raised to its position index (starting from 0 on the right) and sum the results. For example, 1011 = 1×8 + 0×4 + 1×2 + 1×1 = 11.

Decimal to Hexadecimal: Divide the decimal number by 16, recording the remainder. Continue dividing the quotient by 16 until you reach 0. The hexadecimal number is the remainders read from last to first, using A-F for values 10-15.

Other Bases: The same process applies for any base. Use repeated division for conversion from decimal, and positional multiplication for conversion to decimal.

Quiz Tips: Double-check your calculations, especially when converting to and from bases above 10 (where letters are used). Practice with small numbers to build confidence.

Storage Units

This section explains units of digital storage and how to convert between them.

Units of Storage: The basic unit is the bit (b), which can be 0 or 1. 8 bits make a byte (B). Larger units include kilobyte (KB, 1024 bytes), megabyte (MB, 1024 KB), gigabyte (GB, 1024 MB), and so on. Sometimes, 1000 is used instead of 1024 for conversions (especially for hard drives).

Unit Conversion: To convert between units, multiply or divide by 1024 (or 1000, depending on the context). For example, 2048 bytes = 2 KB (since 2048 ÷ 1024 = 2).

Sound, Image, and Text Files: The size of a file depends on its type and properties. For sound files, size = sample rate × bit depth × channels × duration. For images, size = width × height × color depth. For text files, size = number of characters × bytes per character.

Quiz Tips: Read the question carefully to determine which formula to use. Pay attention to units and always show your working for conversions.

Non-Integer Binary

This section covers fixed point and floating point binary representations.

Fixed Point: In fixed point representation, a certain number of bits are used for the integer part and the rest for the fractional part. For example, in 8 bits with 4 integer and 4 fractional bits, 1100.1010 means 12.625 in decimal (12 + 0.5 + 0.125).

Floating Point: For two's complement floating point, the first digit (bit) represents -1, and the rest represent positive fractional values. For example, in a 4-bit two's complement fraction, 1010 means -1 + 0 + 0.25 + 0 = -0.75. This is different from sign-magnitude, and allows negative values by interpreting the first bit as negative.

Operations: Addition and subtraction require aligning the binary points (for fixed point) or exponents (for floating point). Normalisation shifting the mantissa so that the first and second digit of the mantissa are not the same, then adjusting the exponent appropriately.

Binary to Base 10: To convert a binary fraction to decimal, sum each bit multiplied by 2 raised to its negative position index (e.g., 0.101 = 1×0.5 + 0×0.25 + 1×0.125 = 0.625). For two's complement, remember the first bit is negative. For floating point, first convert the mantissa (using two's complement if needed), then multiply the result by 2 raised to the exponent value.

Quiz Tips: Carefully identify the type of representation and follow the correct steps for each operation. Practice converting between binary and decimal for both integer and fractional values.
